# Gemini에게 PPT 발표 대본 요청하기

**사용 방법**  
1. 아래 "Gemini에게 보낼 프롬프트" 블록을 **전부 복사**한다.  
2. Gemini에 가서 **PBL_HACATHON.pptx 파일을 업로드**하거나, "슬라이드가 N장이고 제목은 ~이다"처럼 간단히 설명한다.  
3. 복사한 프롬프트를 붙여 넣고 **대본 생성**을 요청한다.  
4. 필요하면 "한국어로", "영어로", "5분이 아니라 3분으로"처럼 조건을 추가한다.

---

## Gemini에게 보낼 프롬프트 (복사용)

```
저는 PBL 해커톤 발표용 PPT(PBL_HACATHON.pptx)를 가지고 있고, 이 PPT에 맞춰 **발표 대본(스크립트)**을 만들어 달라고 부탁드립니다.

### 프로젝트 요약
- **서비스명**: My Networking Agent  
- **한 줄 설명**: 명함을 수집하고, 채팅으로 "React 개발자 구해요"처럼 요청하면 AI가 수집 명함 중에서 맞는 사람을 골라주고, 연락 메시지 초안까지 작성해 주는 앱입니다.  
- **발표 대상**: PBL 해커톤 심사/참가자  
- **발표 시간**: 5분 (슬라이드 수에 맞춰 구간별로 나눠 주세요)

### 슬라이드별로 넣을 핵심 내용 (이 순서/내용을 반영해서 대본 작성해 주세요)

1. **표지**  
   - 제목: My Networking Agent  
   - 부제: 명함 수집부터 AI 추천·연락 초안까지, 한 앱에서  

2. **문제 정의**  
   - 명함은 모으기만 하고 잘 활용되지 않음  
   - "누구한테 연락할지, 뭐라고 쓸지" 고민  
   - 수집한 명함 + 대화만으로 맞춤 추천·메시지 초안을 받고 싶다는 니즈  

3. **솔루션 한 줄**  
   - 수집 명함 + 자연어 요청 → AI가 조건에 맞는 사람 선정 + 연락 메시지 초안 작성  
   - Find(명함 발견) → 수집 → Ask(채팅)에서 말하면 끝  

4. **핵심 기능 3가지**  
   - 명함 발견·수집 (Find / Collect)  
   - AI 에이전트 Ask: 멀티 에이전트, **같은 작업에 다른 모델(GPT-4o-mini vs GPT-3.5)로 다른 아이디어** 도출 후 총괄이 종합·타협. 감시·관리·문제 해결.  
   - 내 명함 관리 (홈, QR/공유)  

5. **기술 스택 & 아키텍처**  
   - Frontend: React, Vite, TypeScript, Tailwind, Capacitor  
   - Backend & DB: Supabase (Auth, Database, Edge Functions)  
   - AI Layer: 멀티 에이전트, GPT-4o-mini + GPT-3.5 병렬 사용, 다른 모델로 다른 아이디어 → 종합·타협  

6. **멀티 에이전트 플로우 (Ask)**  
   - 총괄 에이전트가 하위 에이전트 순차 호출  
   - 같은 작업에 **서로 다른 모델** 사용 → **서로 다른 아이디어** 제출 → 총괄이 종합·타협·다음 단계  
   - 플로우: Analyze Request → (선택) Suggest Idea / Suggest Tech → Context Matcher → Message Architect → Suggest Contact  
   - 차별점: 한 모델이 아니라 다른 모델이 다른 아이디어를 내고, 논의·타협으로 더 나은 결과  

7. **차별점 & 다음 단계**  
   - 차별점 1: 같은 작업에 **서로 다른 모델**로 **서로 다른 아이디어** 도출 → 종합·타협 (다양성 확보)  
   - 차별점 2: 멀티 에이전트 상호 감시·관리  
   - 차별점 3: 명함 수집과 "누구에게 뭐라고 연락할지"를 한 흐름으로 연결  
   - 다음: 명함 데이터 확대, 추천 정확도·연락 발송 UX 개선  

8. **마무리**  
   - My Networking Agent — 수집한 명함, 이제 제대로 써보세요  
   - 감사 인사, QR 또는 데모 링크 안내  

### 대본 형식 요청
- **슬라이드 번호별**로 구간을 나눠 주세요. 예: [슬라이드 1], [슬라이드 2], …  
- 각 구간마다 **대략 몇 초~몇 분** 말하는지 표시해 주세요. (총 5분 이내)  
- **말할 내용**을 그대로 읽을 수 있는 문장으로 작성해 주세요. (발표 톤, 자연스러운 연결)  
- **언어**: (원하는 것 하나만 남기고 나머지 삭제 후 보내세요)  
  - [ ] 한국어  
  - [ ] 영어  
- PPT에 슬라이드가 8장이 아니면, "제 PPT는 N장이고 제목은 ~, ~, ~ …" 이렇게 알려주시면 그에 맞춰 대본을 다시 짜 드릴 수 있다고 안내해 주세요.
```

## 참고: PPT를 Gemini에 넣는 방법

- **파일 업로드**: Gemini 대화창에서 PBL_HACATHON.pptx를 첨부한 뒤, 위 프롬프트를 붙여 넣으면 슬라이드 순서와 내용을 참고해 대본을 맞춰 줄 가능성이 높습니다.  
- **슬라이드 수/제목만 알려주기**: "슬라이드는 8장이고, 1표지 2문제정의 3솔루션 4핵심기능 5기술스택 6에이전트플로우 7차별점 8마무리예요"처럼 적어 주고 같은 프롬프트를 보내도 됩니다.

이 파일은 `docs/gemini-대본-요청.md`에 저장되어 있습니다. Gemini에 보낼 때는 위 "Gemini에게 보낼 프롬프트" 블록만 복사해서 쓰면 됩니다.
