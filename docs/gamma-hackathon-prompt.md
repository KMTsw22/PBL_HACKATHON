# Gamma PPT 해커톤 발표용 프롬프트

아래 블록 전체를 복사해서 Gamma에 붙여넣고, "AI가 프레젠테이션 생성" 또는 유사 옵션으로 넣으면 됩니다.

---

## [아래부터 복사]

**해커톤 발표용 피칭 덱을 만들어줘. 한국어로, 슬라이드 10~12장 분량.**

**프로젝트명:** My Networking Agent (네트워킹 에이전트)

**한 줄 요약:**  
명함을 수집·관리하고, AI 에이전트가 “기획 → 기술 → 인맥 선정 → 메시지 초안 → 연락 발송”까지 한 번에 이어주는 인맥/팀 구성·연락 추천 앱.

**해결하는 문제:**  
- 명함은 모았는데 누구한테 뭘 어떻게 연락할지 막막함.  
- 팀 빌딩·구인·구직·취미 매칭할 때 “우리 수집 명함 중에서 누가 맞지?”를 사람이 일일이 찾기 어려움.  
- 기획·기술 스택까지 정한 뒤 “이 조건에 맞는 사람 찾고, 연락 메시지까지 써줘”를 한 플로우로 해결하고 싶음.

**주요 기능 (5가지):**  
1. **Find** – 명함 탐색·수집 (다양한 직업·전공·언어·프레임워크 시드 데이터 지원)  
2. **Collect** – 내가 수집한 명함 관리·평가(점수·메모)  
3. **Messages** – 1:1 DM (인스타 디엠 스타일: 채팅 목록 + 스레드)  
4. **Ask** – AI 에이전트로 “팀 구성·연락” 요청 → 기획·기술·인맥 선정·메시지 초안·앱 내 DM 발송까지 자동  
5. **Settings** – 프로필·데모용 “전체 명함 일괄 수집” 등

**Ask 에이전트 구조 (핵심 차별점):**  
- **총괄 에이전트(오케스트레이터)**가 사용자 한 마디를 받고, 목적에 맞는 **서브 에이전트**를 순서대로 호출.  
- 모든 추천은 **사용자가 수집한 명함만** 대상 (외부 인재 풀 없음).  
- 서브 에이전트: 기획 에이전트 → 개발(기술 스택) 에이전트 → Data Miner(조건 추출) → **Context Matcher(인원 선정)** → Message Architect(개인화 메시지 초안) → **Contact 에이전트(앱 내 DM 자동 발송)**.  
- **Context Matcher**는 GPT-4o-mini와 GPT-3.5 두 모델이 각각 선정한 결과를 보여주고, 종합해서 최종 인원·메시지로 이어짐 (다양한 관점 반영).  
- 플로우 예: “당구 어플 만들고 싶어” → 기획 정리 → 기술 스택 추천 → 조건 추출 → 수집 명함에서 맞는 사람 선정 → 인원별 연락 문구 생성 → DM 발송.

**기술 스택:**  
- Frontend: React, Vite, TypeScript, Tailwind CSS, Capacitor(모바일)  
- Backend: Supabase (Auth, DB, Storage, Edge Functions, Realtime)  
- AI: OpenAI (오케스트레이터 + 서브 에이전트, 스트리밍 응답)

**타겟 사용자:**  
- 네트워킹·커리어·사이드 프로젝트로 인맥을 쌓는 사람  
- 해커톤·스타트업 팀 빌딩, 구직·구인, 취미 매칭을 한 플랫폼에서 하고 싶은 사람  

**데모 시나리오:**  
1) Find에서 명함 수집 → 2) Ask에서 “React 개발자 구해요” 또는 “당구 앱 만들고 싶어” 입력 → 3) 에이전트가 기획·기술·선정 인원·연락 초안까지 보여줌 → 4) “연락 보내기”로 앱 내 DM 발송.

**마무리 슬라이드:**  
- “명함 수집부터 AI 추천·연락까지, 한 곳에서.”  
- 향후: 다중 LLM(OpenAI + Gemini) 토론형 에이전트, 더 많은 시드 데이터·직군 확장.

발표 시 유의사항: 슬라이드마다 문장은 짧고 핵심만. 숫자·플로우는 불릿과 다이어그램으로 정리해줘.
